# ソフトウェア工学講義内容

## ソフトウェアの特徴

ソフトウェアは３つの要素からなる。
1. 実行される命令語群
2. プログラムが情報を扱うためのデータ構造
3. 操作・使用法を記述した情報
物理的には劣化しないが、環境変化への未対応により**悪化**する性質を持つ。

## ソフトウェア工学の必要性

以下の4つの背景から必要とされる。

1. **大規模化・複雑化**：部品化・オブジェクト化による効率化が有効
2. **IT人材不足**：量的不足＋質的不足
3. **社会性と求められるミッション**：インフラとしてのソフトウェアの継続性
4. **不確実性の増大**：利用の多様化・意図しない活用


## Hyrumの法則とコスト構造

あるAPIに十分な数のユーザーがいるとき、作られたシステムが持つあらゆる観察可能な挙動に関して、それに依存するユーザーが出てくるものである。

**コストの分類**
- 財務コスト
- 資源コスト
- 人件コスト
- 処理コスト
- 機会コスト
- 社会コスト

## ソフトウェアライフサイクル

- **要件定義**：実現する／しない機能の明確化、関係者間の合意形成
- **開発**：内製か外部委託（オフショア開発を含む）かの選択
- **テスト**：全数テストは不可能なため、範囲・深度の判断が必要
- **運用・保守**：属人化リスクへの対処
- **ドキュメンテーション**：非同期コミュニケーション手段としての文書化

## プロジェクトマネジメントの考え方

- プロジェクトは**有期性**と**独自性**を持つ仕事
- 目標から逆算する**バックキャスティング**思考が基本
- **QCD**はトレードオフの関係にあり、システムの性質で優先順位が変わる

**ソフトウェアの評価指標**
- コード物量
- バグ発生率
- ファンクションポイント法
- 使い勝手

## 開発プロセスモデル

| モデル | 特徴 |
|---|---|
| ウォーターフォール型 | 工程を順に進める。進捗管理は容易だが後工程にしわ寄せが集中 |
| スパイラルモデル | プロトタイプとフィードバックの反復 |
| 反復型 | 機能単位での積み上げ開発 |
| アジャイル | 変化への迅速な対応を重視 

## 実践的な計画立案手法：WBS

プロジェクトの成果物達成に必要な作業を階層的に分解する手法。

- スコープの明確化
- 作業の網羅的な洗い出し
- 全体管理の可視化
